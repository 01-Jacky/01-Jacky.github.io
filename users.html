User Test

Link to Question Test

A question test was conducted to gauge which part of the site was most important and what can be improved. Feedback suggested the experience section was most important and the overall design of the site was between neutral and good. User have reiterated a desire for a project section.

A click test was conducted to ensure the "project summary" button on the experience page was easy to find. Result was most users succeeded in getting to the right place.

Summary of Finding

Most users found the sections of the site to be obvious and easy to get to. They did not have major complain about the look of the site but noted that it could be better. Users did not find any bugs with the function or look of the sites. Users were happy with the feature of being able to click on a button in one of my experience to learn more about what I did during that particular internship experience.

The main concern from user interviews and cognitive walkthrough were the lack of a project section. The users that go on my site are typically concerned about my skill level. While the existing experience page and skills page help give user some idea of my skill level, the lack of a project section was reiterated throughout user interview, competitor analysis, and user testing. Users expects and highly value the a project section that briefly describe a project I worked on, and link to the source code.

The other concern was that while most users found the design to be easy to user and navigate, they also found the aesthetic to be lackluster. Competitor analysis suggested the site can benefit from better use of color schemes and interesting imagery. Some users commented that some sections of the site were also too text heavy and wish some of the text can be replaced or supplemented with some images.

Incorporation of Findings

Based on user research and testing, there plan is to take these two immediate action that will greatly improve the value of the site to the user:

1) Add a new project section. From user interviews and cognitive walkthrough, users highly value being able to see some projects the person has worked on and being able to view the source code to them. In some ways they were disappointed that it wasn't there. In the future, this section should be added and highlight some interesting projects, sample image(s) of the project in action, a short description, and most importantly link the source code of it. This feature will bring the portfolio much closer to the level of the some of the portfolio analyzed during competitors analysis.

2) Improve the design of the site by changing the color scheme and adding more images. While users found current design and look to be satisfactory, it was also very uninspired. It is missing the wow factor that some of the competitor's website has when you first land on it. Users also complained how sections are too text heavy sections, and so in the future the text should be reduced and/or accompanied by related images. The site will also be redesigned to take on more interesting color schemes and/or layout in order to deliver that wow factor when the user first sees it.

Incorporating the 2 changes (Future):

1) A new project section will be added to the nav bar. This new section should highlight at least 3 projects with link to the source code on github. If it is a web project, an additional link should be placed to see the project in action. Each project should an interesting image that gives an idea of what the project is about in addition to a short description. As I complete more projects, perhaps setup each project in a grid similar to the skill section.

2) Redesign: First use a color scheme designer to identify a more interesting color scheme. A website like paletton will help with the process. Then explore a new layout. Some idea include a one page layout, or the golden ratio layout from the ibm design language. Also refactor the text on the About page to be more concise.
